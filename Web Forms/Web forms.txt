<< installation >>

- Open the Visual Studio Installer from your Start menu.
- Find your installed version of Visual Studio 2026 and select the Modify button.
- In the Workloads tab, ensure the "ASP.NET and web development" workload is checked.
- Go to the Individual components tab.
- Search for and check the box for ".NET Framework project and item templates".
- Click Modify to install the selected components. You may need to restart Visual Studio after the installation is complete.

<< Project Structure >>

- Default.aspx → Home page Web Form.

- Site.Master → Master page (common layout like header/footer).

- Web.config → Configuration settings.
- Scripts → JS files.
- App_Code → Optional, you can place helper classes here.
- App_Data → Database files if using local databases.

<< page directive >>

- Page Directive in ASP.NET Web Forms, and it appears at the very top of every .aspx page
- Think of it as a configuration header that tells ASP.NET how to handle this page.

- <%@ Page Title="Home Page" Language="C#" MasterPageFile="~/Site.Master" AutoEventWireup="true" CodeBehind="Default.aspx.cs" Inherits="WebForm0._Default" %>

   - <%@ Page ... %>: This tells ASP.NET: “This is a Web Form page”., its called directive
   
   - Title="Home Page" : Sets the title of the page, which appears in the browser tab.
   
   - Language="C#" : Specifies the programming language used in the code-behind.
   
   - MasterPageFile="~/Site.Master" : Indicates this page uses a Master Page.
 
   - AutoEventWireup="true"
	- If true, ASP.NET automatically connects events like Page_Load to their methods in the code-behind.
	- If false, you would need to manually attach events, like --> this.Load += new System.EventHandler(this.Page_Load);

   - CodeBehind="Default.aspx.cs"
	- Points to the C# file that contains the server-side logic for this page.
	
   - Inherits="WebForm0._Default" 
	- Specifies the fully-qualified class name of the code-behind --> public partial class _Default : Page

<< <asp:Content>   >>

- this is a server-side control.
- It defines the content for a Master Page placeholder.
- Every Web Form that uses a Master Page will have one or more <asp:Content> sections.
- Everything between the opening <asp:Content> and closing </asp:Content> will be rendered inside the placeholder of the Master Page.

<asp:Content ID="BodyContent" ContentPlaceHolderID="MainContent" runat="server">

- ID="BodyContent"
	- This is just a unique identifier for this <asp:Content> block.
	- You can reference it in code if needed (rarely required).

- ContentPlaceHolderID="MainContent"
	- This content belongs inside the ContentPlaceHolder element with ID = "MainContent" in the Master Page.
	- Example in Site.Master:
		<asp:ContentPlaceHolder ID="MainContent" runat="server">
    			<!-- Child pages will fill this -->
		</asp:ContentPlaceHolder>

- runat="server"
	- This tells ASP.NET that this control is server-side.
	- It allows you to manipulate it in C# code-behind if needed.
