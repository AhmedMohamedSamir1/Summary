what is the difference between object and var and dynamic ?

ال object :  هو ref DT وهو ال base type لـكل الدوت نت DT سواء كانت Value type  او ref type ,علشان كدا اقدر اخزن فيها اى value لاى DT 
	: وبحتاج اعمل casting لو حبيت اعمل operation على ال Value المتخزنة فى ال object ده 
         :  وبيحصل boxing و unboxing لما assign و retrive فالــيو DT لل object وده مش حلو ك performance لان انا بحط ال Value DT فى ال heap فده يعتبر expensive مقارنة لو انا شغال فى ال stack انا هنا بحتاج
      additional info  لكل boxed value وده يعتبر Memory Overhead 
     :  Useful for scenarios when you don't know the exact type until runtime 
     : int number = (int)Convert.ChangeType(obj, typeof(int))
  
	
ال var عبارة عن keyword : بتخلى ال Compiler يتعرف على نوع ال DT من خلال ال Right hand side expression ,
		 لازم اعمل initialization او expression وانا ب declare ال variable باستخدام ال var
		 يعنى بيحصل Type checking is done at compile time. 
		وبستخدم لما أكون عايز اسهل عليا كتابة ال complex type declarations وممكن يكون عيب ان كدا الكود ممكن ميكونش Readable

ال dynamic   :  ال type checking بيحصل فى ال runtime مش فى ال compile-time
	  : you can assign a value of one type to a dynamic variable and later assign a value of a different type 	
	  : Incorrect Method Invocation --> على سبيل المثال 
	  : dynamic obj = "Hello, World	
          : obj.NonExistentMethod(); // Runtime error: 'string' does not contain a definition for 'NonExistentMethod   	
	  : does not require explicit casting. This is one of the main features that distinguishes dynamic from object. 
	  : When you use a dynamic variable, the C# runtime handles the type resolution and any necessary casting for you

const, static, readonly

const
  - The value of a const field is set at compile time and cannot be changed thereafter.
  - fields are implicitly static, so you do not need to use the static keyword with them.
  - only Primitive Types and Strings: You can only use const with primitive types (e.g., int, double) and strings.
 
readonly
  - Runtime Constant:The value of a readonly field can be set either at the time of declaration or in the constructor of the class then,it cannot be changed
  - readonly fields can be static or non-static
  - Can be used with any type

static 
  - fields or methods belong to the type itself 
  - fields can be initialized in a static constructor, which is called once when the class is first accessed

what is the difference between app.Run, app.Use, app.Map

app.run(): it does not pass the request to the next middleware component in the pipeline. it is used to add terminal middleware at the end of the middleware pipeline
	ensuring that all requests are handled by the pipelines

app.Use(): add middleware to middleware pipeline, can perform operations before and after passing request the next middleware
app.Map(): It allows you to define separate middleware pipeline branch for different URL prefixes.
