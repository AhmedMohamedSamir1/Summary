Serialization: 
	- is the process of converting an object into a format that can be stored or transmitted
	- It’s like turning a complex object (with data, fields, structure) into a string or binary data
          so you can save it, send it, or store it — and then deserialize it back later.

- JSON.stringify(JS_Object)
	- It serializes the object into a JSON string. 		
	- This is useful when you need to send data over the network, store it in a file

- JSON.parse(text) 
	- takes a JSON string and transforms it into a JavaScript object.

----- Object.defineProperties()
		
	- used to add properties to an object however it gives you fine-grained control over how the property behaves.

	- Object.defineProperties(obj, props)
		obj: The object you want to define new properties on.
		props: An object containing one or more property descriptors.

 	- For example:
	   let user = {
  		userName: "ahmed",
  		age: 22,
  		doubleAge: function(){ return this.age * 2; }
	   }	
	   Object.defineProperty(user, "email", {
  		value: "ahmed@gmail.com",
  		writable: false,      // can't change it
  		enumerable: true,     // will show in loops like for...in
  		configurable: false   // can't delete or redefine
	   });

---- object prototype.

   - Every JavaScript object has an internal link to another object called its prototype.
   - When you try to access a property or method on an object:
	- JavaScript first looks at the object itself.
	- If it doesn't find it, it looks at the object's prototype.
	- This continues up the prototype chain until it either finds the property or reaches the end (null).
	
   - You can access an object's prototype using
	- Object.getPrototypeOf(obj);
	- console.log(obj.__proto__);

   - You can set the prototype of objec
   	- Object.setPrototypeOf(source, { sayHi: () => "Hi" });

--- Object.assign()
	- copies the values of all enumerable own properties from one or more source objects to a target object. It returns the target object.
	- the prototype  is not copied , Only the enumerable properties  
        - Object.assign(target, ...sources);
		- target: the object to receive the properties.
		- sources: one or more objects whose properties will be copied into the target.


	let info1 = { name: "ahmed", age: 22 }
   	let info2 = { email: "ahmed@gmail.com" }
	
	Object.assign(info2,  info1);
	console.log(info2); // {email: 'ahmed@gmail.com', name: 'ahmed', age: 22}

--- Object Destructuring in JavaScript
      
      ---  Ex1
	 const user = {
  	   name: "Ahmed",
  	   age: 25,
  	   email: "ahmed@example.com"
	 };

	 // Destructuring
	 const { name, age } = user;

	 console.log(name); // "Ahmed"
	 console.log(age);  // 25


    --- You can rename the extracted variables:
	  const { name: userName, age: userAge } = user;
	  console.log(userName); // "Ahmed"
	  console.log(userAge);  // 25

     --- If the property doesn’t exist in the object, you can assign a default value:
	 const { city = "Cairo" } = user;
	 console.log(city); // "Cairo" (default)

     --- You can also destructure nested objects:
	 const person = {
  	   name: "Ahmed",
  	   address: {
    		city: "Giza",
    		zip: "12345"
  	  }
	 };

	const { address: { city } } = person;
	console.log(city); // "Giza"


   --- Using in Function Parameters
	function printUser({ name, age }) {
  		console.log(`Name: ${name}, Age: ${age}`);
	}
	printUser(user); // "Name: Ahmed, Age: 25"
