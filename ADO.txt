ADO modes :
	connected mode : 
		
		- used with real time application because you want the data to be updated
		
		- any changes made to the data by your application or other users/processes are immediately reflected.
		
	
	disconnected mode:	
		- your application connects to the database, retrieves data into a local dataset or data table [memory], 
		  and then disconnects from the database.
		
		- Once the data is fetched into memory, the connection to the database is closed, 
		  and your application can work with the data locally (offline)
		
		- Disconnected mode is useful when you need to work with data offline or perform complex data manipulation
		  without directly affecting the database until you're ready to commit changes.

		- When working in disconnected mode, you can make changes to the data in memory, and later you can use a DataAdapter
		  to reconcile those changes back to the database, changes can be insert, update, delete
	
		- The process of reconciling changes back to the database ensures that the changes made locally are reflected 
		  in the original database, thus maintaining data integrity and consistency between the application and the database

<< To add an application configuration file to a C# project >>

1. In Solution Explorer, right-click the project node, and then select Add > New Item.
2. Expand Installed > Visual C# Items.
3. In the middle pane, select the Application Configuration File template.
4. Select the Add button.

<< to add connection string at application configuration file >>

<connectionStrings>
	<add name="ITI" connectionString="Server=.;Database=ITI;Trusted_Connection=True;TrustServerCertificate=True"/>
</connectionStrings>

name: The name used to identify the connection string.
connectionString: The actual connection string containing details such as the server, database name, and authentication options.

(. , which represents the local server)
